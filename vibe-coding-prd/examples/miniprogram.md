## C — Context

- **一句话定位**：让公司团队成员快速报名公司内部活动的微信小程序
- **平台**：微信原生小程序（WXML + WXSS + JS）+ 微信云开发（CloudBase）
- **用户**：公司团队 50 人，内部使用（小程序未公开发布，体验版 + 二维码）
- **工期**：5 天（每天约 2 小时）
- **参考产品**：金数据（功能太重），目标做"3 个字段就能报名"的极简版

---

## S — Scope

### v0.1 必做（MUST）

- 用户能够查看正在进行的活动列表（含标题 + 时间 + 报名人数 / 上限）
- 用户能够点开活动详情查看完整介绍
- 用户能够一键报名（自动获取微信昵称 + openid，不再填表）
- 用户能够取消报名（活动开始前 1 小时之前可取消）
- 用户能够查看自己已报名的活动列表
- 管理员能够创建新活动（标题、详情、时间、人数上限）
- 管理员能够查看每个活动的报名名单（昵称 + 报名时间）
- 管理员能够导出报名名单为 Excel

### v0.1 不做（MUST NOT）

- ❌ 跨平台支付（活动免费），原因：免费团建活动
- ❌ 客服 / IM 功能，原因：用公司微信群代替
- ❌ 活动评价 / 评论，原因：v0.2 再说
- ❌ 推送通知，原因：依赖微信订阅消息授权，v0.2 再加
- ❌ 多语言支持，原因：纯中文团队
- ❌ H5 版本，原因：微信内即用，无需 H5

### v0.2 计划（LATER）

- 微信订阅消息提醒（活动前 1 天）
- 活动签到（扫码）
- 活动评价

### v0.3 远期（IDEAS）

- 跨公司活动（多租户）
- 报名审核流程

## P — Specification

### P.1 视觉规范

- **主色**：`#07c160`（微信绿，符合微信生态）
- **背景色**：`#f7f7f7`
- **卡片背景**：`#ffffff`
- **主文字**：`#191919`
- **次文字**：`#888888`
- **错误红**：`#fa5151`
- **字体**：使用微信默认字体（不自定义 font-family）
- **字号**：标题 `32rpx`，正文 `28rpx`，次要信息 `24rpx`
- **圆角**：卡片 `16rpx`，按钮 `48rpx`（pill）
- **间距**：卡片间 `24rpx`，卡片内边距 `32rpx`

### P.2 行为规范

- 首次进入小程序 → 自动调 `wx.login` 获取 openid（无需用户手动登录）
- 用户身份判断：openid 在 `admins` 集合中 → 显示"管理员"入口，否则普通用户
- 点击「报名」按钮 → 调 `wx.getUserProfile` 获取头像昵称 → 写入云数据库 → 按钮变"已报名"
- 点击「取消报名」→ 弹确认浮窗 → 确认后从数据库删除 → 按钮变回"立即报名"
- 活动人数到上限 → 报名按钮变灰显"已报满"
- 活动开始时间 < 1 小时 → 取消按钮变灰显"无法取消"
- 管理员创建活动 → 表单页 → 提交后跳转活动列表
- 管理员导出 → 调云函数生成 Excel → 通过 `wx.openDocument` 打开

### P.3 数据规范

```javascript
// 云数据库集合：events（活动）
{
  _id: string,
  title: string,           // 标题
  description: string,     // 详情
  startTime: Date,         // 开始时间
  capacity: number,        // 人数上限
  enrolledCount: number,   // 已报名数（冗余字段，方便查询）
  createdBy: string,       // 创建者 openid
  createdAt: Date
}

// 云数据库集合：enrollments（报名记录）
{
  _id: string,
  eventId: string,
  openid: string,
  nickname: string,
  avatarUrl: string,
  enrolledAt: Date
}

// 云数据库集合：admins（管理员）
{
  _id: string,
  openid: string,
  addedAt: Date
}

// 业务约束：
// - (eventId, openid) 联合唯一（防止重复报名）
// - enrolledCount 必须 ≤ capacity
// - 活动开始 1 小时内不能取消报名
```

### P.4 技术规范

- **框架**：微信原生（**不**用 Taro / uni-app / Mpvue）
- **后端**：微信云开发（CloudBase）—— **不**自建后端
- **数据库**：云开发自带的 NoSQL 数据库
- **云函数**：用 Node.js 18 写
- **不用**：Vant / iView / 任何第三方 UI 组件库
- **导出 Excel**：云函数用 `xlsx` 库生成
- **文件结构**：

```

├── miniprogram/
│   ├── pages/
│   │   ├── index/         # 活动列表

│   │   └── admin/
│   │       ├── create/    # 创建活动
│   │       └── manage/    # 管理报名
│   ├── components/
│   │   └── event-card/    # 活动卡片
│   ├── utils/

│   ├── app.js
│   ├── app.json
│   └── app.wxss
```

## G — Guardrails

### 禁止事项

- ❌ **不要换框架到 Taro / uni-app** / 原因：原生最稳，避免编译问题
- ❌ **不要自建后端服务器** / 原因：云开发够用且零运维
- ❌ **不要引入 Vant / iView 等组件库** / 原因：原生已能满足，加库会增加体积
- ❌ **不要存用户敏感信息**（手机号、身份证等） / 原因：无业务必要，且涉及合规
- ❌ **不要用 `wx.getUserInfo`** / 原因：已废弃，必须用 `wx.getUserProfile`
- ❌ **不要在前端存管理员列表** / 原因：管理员判断必须走云函数
